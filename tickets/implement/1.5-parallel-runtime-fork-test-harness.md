## Goal

Land three defensive tests before any consumer of `ParallelDriver` ships, so that future plan nodes / emitters / context fields can't silently violate the fork contract. The harness is purely additive тАФ no production behavior changes тАФ but it turns every existing test into a fork-contract audit.

This is a hardening pass between #1 (`parallel-driver-context-fork`, complete) and #2 (`parallel-eager-prefetch-node`). The user explicitly chose the test-only approach over a heavier `TableContextStore` encapsulation: if strict-fork mode flags real violations later, *then* we upgrade the encapsulation; until then, tests cover the risk surface.

## Why this lands before #2тАУ#5

The fork contract has three implicit invariants today:

1. Every field of `RuntimeContext` has a fork policy (forked / shared / frozen / cooperative). The driver enumerates today's fields; a new field added without thinking is silently shared by reference.
2. Mutable fork-isolated maps (`tableContexts`, `context`) must not be mutated by the *parent* while forks are alive тАФ fork snapshots are taken at fork-time.
3. Only specific emit sites legitimately mutate `tableContexts` after context creation. Today: only `runtime/emit/recursive-cte.ts:106-129`.

None of these are checked anywhere. Landing the consumers (`EagerPrefetchNode`, `FanOutLookupJoin`, etc.) before the checks means each consumer carries the risk of activating a latent violation. Landing the checks first means the first consumer ships into a guarded environment.

## The three tests

### Test 1 тАФ Pinned-keys drift

`packages/quereus/test/runtime/fork-contract.spec.ts` (new file). A single test that pins the field set of `RuntimeContext` against a hardcoded list, with each field tagged by fork policy:

```ts
const EXPECTED_FORK_POLICY = {
  db:                'shared-frozen',     // immutable reference
  stmt:              'shared-frozen',
  params:            'shared-frozen',
  context:           'forked',            // independent per branch
  tableContexts:     'forked',
  tracer:            'shared-sink',       // write-only instrumentation
  activeConnection:  'shared-cooperative', // vtab concurrencyMode declares safety
  enableMetrics:     'shared-frozen',
  contextTracker:    'shared-sink',
  planStack:         'shared-sink',
} as const;
```

The test extracts `keyof RuntimeContext` at the type level (via a helper using `satisfies Record<keyof RuntimeContext, ForkPolicy>`) and asserts equality. New field on `RuntimeContext` тЖТ TypeScript compile error in the test тЖТ author must declare policy тЖТ forces the conversation to happen.

This is the cheapest, highest-leverage check. ~30 lines.

### Test 2 тАФ Mutation-site allowlist

Same spec file. A test that grep-walks the source tree for every site that mutates `RuntimeContext.tableContexts` and `RuntimeContext.context` *after construction*. Construction sites (`new Map()`, `new Map(rctx.tableContexts)`) are excluded тАФ they appear in `core/statement.ts`, `core/database.ts`, `core/database-assertions.ts`, `runtime/deferred-constraint-queue.ts`, `planner/analysis/const-evaluator.ts`, and `runtime/parallel-driver.ts` (the fork itself).

Mutation sites are matches like `tableContexts.set(`, `tableContexts.delete(`. The test compares the found set against an allowlist:

```ts
const TABLE_CONTEXTS_MUTATION_ALLOWLIST = new Set([
  'packages/quereus/src/runtime/emit/recursive-cte.ts',
  // any future legitimate site lands here only after deliberate review
]);
```

For `context` (RowContextMap) mutations: the only legitimate writer is `context-helpers.ts:createRowSlot`. Every other site uses `createRowSlot`'s return value (its `set`/`clear`); the allowlist tracks the helper itself.

Implementation: walk `packages/quereus/src/` via `glob` + `readFileSync` + regex. New file with `tableContexts.set(` тЖТ test fails тЖТ author adds the path to the allowlist after reading this ticket's contract section. The git-blame trail makes the audit explicit.

~40 lines.

### Test 3 тАФ Strict-fork test mode

The headline check. A test-only mode that wraps `tableContexts` and `context` on every `RuntimeContext` with a `Proxy` that:

- Tracks an `activeForks` counter (incremented by `ParallelDriver` on `fork()`, decremented when `drive()` exits).
- On any `set` / `delete` while `activeForks > 0`, throws a `QuereusError` with a clear message: `"strict-fork: parent context mutated tableContexts while N forks are active. This is a fork-contract violation. See docs/runtime.md ┬з Parallel runtime fork contract."`.

Gated by an env flag: `QUEREUS_FORK_STRICT=1`. Production code path is unchanged when the flag is off тАФ only a single `if (process.env.QUEREUS_FORK_STRICT) ...` check at context-construction time.

A new yarn script runs the full quereus test suite under the flag:

```json
"test:fork-strict": "QUEREUS_FORK_STRICT=1 yarn test"
```

Wire it into the package's regular CI matrix (or, if CI is out of scope here, document the manual invocation in the review handoff so it gets run before the next ticket starts).

The counter lives on `ParallelDriver` and is bumped in `drive()` rather than `fork()` тАФ `fork()` may be called speculatively without a subsequent `drive()`. Bump on `drive()` entry, drop on its `finally`. The proxy reads the counter via a back-reference set at fork time.

Implementation cost: ~80 lines split between `parallel-driver.ts` (the counter + proxy install helpers, all gated) and `context-helpers.ts` (a `createStrictRowContextMap` wrapper). All production-path calls are no-ops when the flag is unset.

## Three things to verify before declaring done

- `yarn test` (flag unset) passes тАФ production path untouched, no slowdown, no behavior change.
- `yarn test:fork-strict` passes тАФ the existing suite contains no current violations. If it *fails*, you've found a real bug in current code; fix it (likely in `recursive-cte.ts` via tighter fork-scope, but read the failure first).
- Add a unit test in `fork-contract.spec.ts` that *deliberately* constructs a violation (fork; then mutate parent `tableContexts`; expect throw under strict mode) тАФ proves the harness has teeth.
